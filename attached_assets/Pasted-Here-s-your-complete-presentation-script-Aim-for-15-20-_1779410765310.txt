Show code — getDashboard():

$revenue = $db->query(
  "SELECT COALESCE(SUM(total_amount),0) AS rev
   FROM orders WHERE status != 'Cancelled'"
)->fetch()['rev'];
$pending = $db->query(
  "SELECT COUNT(*) AS c FROM orders WHERE status = 'Pending'"
)->fetch()['c'];
$products = $db->query(
  "SELECT COUNT(*) AS c FROM products"
)->fetch()['c'];
// Top selling products
$db->query(
  "SELECT oi.name, COUNT(*) AS sold
   FROM order_items oi
   JOIN orders o ON oi.order_id = o.id
   WHERE o.status != 'Cancelled'
   GROUP BY oi.name ORDER BY sold DESC LIMIT 5"
);

Say:

"Each stat is its own query — revenue is a SUM of all non-cancelled orders, pending is a COUNT filtered by status, and top products uses a JOIN between order_items and orders grouped by product name."

PART 9 — Admin: Orders Management (1–2 min)
Do: Click Orders in the sidebar. Show the orders list.

Say:

"Admins can view all customer orders and update their status through the fulfillment pipeline."

Show code — adminGetOrders() query:

"SELECT o.*, u.username AS customer,
        json_agg(json_build_object('name',oi.name,'price',oi.price)) AS items
 FROM orders o
 JOIN users u ON o.user_id = u.id
 LEFT JOIN order_items oi ON oi.order_id = o.id
 GROUP BY o.id, u.username
 ORDER BY o.created_at DESC"

Say:

"This query joins orders with users and uses json_agg to aggregate each order's items into a JSON array — so the frontend gets everything it needs in a single query instead of multiple round trips."

Show code — status update:
